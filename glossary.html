EVP Glossary

Terms, Protocols, and Cryptography Concepts

1. Roles & Protocols

EVP Email Verification Protocol

The web browser standard that allows websites to automatically verify email ownership without manual OTPs or magic links.

IdP Identity Provider

The organization hosting the email account (e.g., Google, Yahoo) that issues the cryptographic proof of email ownership.

RP Relying Party

Your website or backend application that relies on the token to verify the user's email address.

2. Tokens, Keys & Standards

EVT Email Verification Token

The core identity token issued by the IdP stating: "This email address is verified."

SD-JWT Selective Disclosure JWT

The signed JWT format used for the EVT. It allows selective disclosure (hiding unneeded user attributes or keeping them as hashes).

KB-JWT Key-Bound JWT

A secondary token generated by the user's browser. It binds the EVT to that specific browser instance so nobody else can steal and reuse it.

JOSE JS Object Signing & Encryption

The overarching standard for JSON-based cryptography (covers JWTs, JWKs, Signatures, and Encryption).

JWK JSON Web Key

A single cryptographic key formatted as a JSON object (contains fields like kty for key type, crv for curve, x for public key data).

JWKS JSON Web Key Set

A collection of JWKs published by the IdP. Your server downloads these public keys to verify the IdP's signature on the EVT.

3. Token Payload Claims & Fields

Field Name Full Name Explanation
iss Issuer The origin of the IdP that generated the token (e.g., https://accounts.google.com).
aud Audience The target origin the token was issued for — must match your website's domain.
nonce Number Used Once A random single-use code generated by your backend form to ensure the token cannot be replayed later.
iat Issued At A Unix timestamp indicating when the token was created.
email_verified Email Verif. Status Boolean flag (true / false) stating whether the IdP has confirmed the user's email.
cnf Confirmation Claim A JSON wrapper claim inside the EVT payload holding the confirmation method details.
cnf.jwk Embedded Public Key The browser's temporary Public Key embedded inside the cnf claim of the EVT by the IdP.
sd_hash Selective Discl. Hash A SHA-256 fingerprint inside the KB-JWT payload that locks the KB-JWT to its specific SD-JWT.

4. Fundamental Cryptography Concepts

Hash (SHA-256)

What it is: A 1-way digital fingerprint of data.

Key Characteristic: Uses NO secret key. Anyone hashing the exact same text with SHA-256 will always get the exact same result string.

Used For

Checking Integrity (sd_hash) to verify data wasn't tampered with or swapped.

Ephemeral Key

What it is: A temporary public/private keypair created by the browser for a single transaction.

Key Characteristic: Thrown away immediately after the form submission completes.

Used For

Proof of Possession. Proves the client submitting the form holds the private key matching the public key sealed inside the EVT.

Digital Signature

What it is: A cryptographic seal produced using a Private Key and verified using a Public Key.


Used For

Authenticity & Non-repudiation. Proves who created the message (e.g., IdP signed EVT; Browser signed KB-JWT).
